# StealthEye Skill — OSS Audit

## Purpose

Assess open-source tools and patterns for public-safe StealthEye Cloud adoption.

## Procedure

1. Prefer patterns over dependencies.
2. Check license posture before adoption.
3. Reject paid-default, local-only, or high-friction requirements.
4. Convert true upgrades into docs, schemas, or small public-safe code.
5. Record adopted patterns in docs or memory only when execution-relevant.

## Rules

1. Do not add bulky frameworks by default.
2. Do not adopt license-risk components as core.
3. Do not add paid service dependencies.
4. Do not create uncontrolled tool chaos.
