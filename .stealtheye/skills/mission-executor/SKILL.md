# StealthEye Skill — Mission Executor

## Purpose

Operate the mission-level execution path without per-file confirmation.

## Procedure

1. Confirm the current mission and approval envelope.
2. Validate intended scope against Active and Decisions.
3. Use high-level mission actions only.
4. Let GitHub Actions prove the branch.
5. Repair only real proof failures.
6. Merge only when green and allowed.

## Boundaries

Stop for secrets, paid compute, private data exposure risk, destructive irreversible action, account permission changes, or unresolved high-impact ambiguity.
