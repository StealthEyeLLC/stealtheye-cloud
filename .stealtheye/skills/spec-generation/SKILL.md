# StealthEye Skill — Spec Generation

## Purpose

Generate build-ready StealthEye Cloud specs and docs that are precise, public-safe, and directly executable by future drops.

## Procedure

1. Anchor to current Decisions and Active state.
2. Enumerate scope, files, schemas, tests, workflows, proof gates, and stop conditions.
3. Avoid generic fluff and vague future claims.
4. Keep public/private boundaries explicit.
5. End with one exact next action.

## Rules

1. Do not weaken architecture for speed.
2. Do not add private strategy to public docs.
3. Do not create conflicting specs.
4. Do not reopen frozen decisions.
