# Mission Approval Envelope

The mission approval envelope converts one explicit user approval into authority for routine public-safe build work inside the accepted scope.

## Covered routine actions

Covered routine actions include safe file creation, safe file updates, branch creation, commit creation, PR creation, CI inspection, exact failure repair, docs/state updates, and merge when green.

## Stop only for true boundaries

Stop only for true boundaries: secrets, paid compute, private data exposure risk, account permission changes, production deployment, database mutation, browser-cookie/session-token automation, destructive irreversible action, or unresolved high-impact ambiguity.

## No weakening clause

The envelope does not authorize weaker validators, skipped workflows, fake proof, placeholder features, relaxed safety boundaries, or merging before green proof.
