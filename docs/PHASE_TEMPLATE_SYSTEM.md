# Phase Template System

## Future phase default contract

Every future phase inherits the S9 default contract:

1. one mission approval by default
2. one coherent repo mutation/drop by default
3. one PR by default
4. one proof wave by default
5. batched repairs by default
6. merge when green by default
7. no placeholders or fake proof
8. no validator weakening
9. merge-aware handoff state
10. final report and next-tab prompt artifact

## Phase checklist

A future phase is not complete until code/contracts, schemas, validators, docs, proof workflow, state files, and final report all agree.
