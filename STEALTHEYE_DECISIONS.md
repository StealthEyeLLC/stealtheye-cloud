# StealthEye Decisions

## Frozen decisions

1. Product name is **StealthEye Cloud**.
2. Public repository is `StealthEyeLLC/stealtheye-cloud`.
3. StealthEye Cloud is no-local by default.
4. ChatGPT 5.5 Thinking is the primary intelligence.
5. GitHub Actions is the default execution/proof body.
6. Browser body is first-class.
7. Rust owns durable core and validators.
8. TypeScript owns browser automation.
9. JSON Schema owns packet contracts.
10. Markdown owns agent-readable state and docs.
11. Use one ChatGPT tab until saturation.
12. Use StealthEye Relay for handoffs.
13. Use StealthEye Seal as the single universal receipt/checkpoint.
14. Use mission-level approvals, not per-file approval.
15. Build in large final-form drops.
16. Do not use Claude, Copilot, Cursor, or soul adapter files.
17. Adopt OSS patterns only when they are real capability/speed/power upgrades with low friction.
18. Use Codex as worker lane only, not as primary brain.
19. Do not claim hidden ChatGPT background autonomy.
20. Public repo must remain public-safe.
21. S0 through S9 are merged green.
22. S6 is **Zero-Trust Cross-Cloud Gateway**.
23. S7 is **First Real Activations**.
24. S8 is **StealthEye Cloud Remediator**.
25. S9 is **One-Drop Build Accelerator**.
26. S10 is **Assistant Optimization Layer**.
27. No placeholder features, fake integrations, or mock success paths labeled real.
28. S6 readiness lanes must ship real contracts, schemas, validators, docs, and proof gates.
29. S7/S8 active lanes must perform the real action and produce proof artifacts.
30. S9 must optimize every future phase toward one mission approval, one coherent repo mutation/drop, one PR, one proof wave, batched repairs, and merge when green.
31. S10 must optimize the assistant/operator layer so ChatGPT uses the S9 rail with better context loading, tool planning, recovery, proof reasoning, low-attention operation, handoff quality, read-only verification, and self-audit.
32. Browser cookie extraction, consumer session rehydration, 2FA bypass, SMS verification automation, and billing-bypass framing are forbidden.
